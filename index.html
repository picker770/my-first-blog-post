My Journey into Web Development

A beginner's reflection and learnings

Why I started learning Web Development?

I’ve always been drawn to creative expression, and web development offers the perfect blend of logic and artistry. The ability to build something functional while designing visually engaging experiences captivated me. Whether it’s crafting clean layouts with CSS or structuring content with HTML, I love how coding turns ideas into tangible, interactive creations. That’s what inspired me to begin this journey.

HTML Fundamentals

I’ve gained a solid understanding of core HTML concepts, including:

  • Structural Elements
    • Proper use of div, span, and semantic tags like header, article, and section .
  • Content Organization
    • (ul, ol, dl), tables (table), and forms (form, input, select) for structured data and user input.
  • Multimedia & Interactivity
    • Embedding images (img), audio/video (audio, video), and dropdown menus (select).
  • Semantic HTML
    • Leveraging elements like nav, footer, and figure for better accessibility and SEO.

CSS Fundamentals

My CSS knowledge includes:

  • Selectors & Specificity
    • Targeting elements using classes, IDs, pseudo-classes (:hover, :focus), and combinators.
  • Layout Techniques
    • Mastering the box model, Flexbox for responsive alignment, and positioning (relative, absolute, fixed).
  • Styling & Effects
    • Customizing typography (font-family, font-size), backgrounds (gradients, overlays), and handling overflow.
  • Maintainability
    • Organizing stylesheets efficiently and understanding file path structures.

Challenges I Faced

  • Browser Inconsistencies
    • Struggled with cross-browser compatibility (e.g., Flexbox gaps in Safari).
    • Learned to use vendor prefixes (-webkit-, -moz-) and progressive enhancement.
  • CSS Specificity Wars
    • Overriding styles unintentionally due to poorly managed selectors.
    • Adopted BEM naming conventions to keep specificity predictable.
  • Responsive Design Hurdles
    • Initially built "desktop-first" layouts that broke on mobile.
    • Mastered media queries and mobile-first workflows.
  • Positioning Puzzles
    • Misused absolute/fixed positioning, causing layout collapses.
    • Practiced with modern layout tools like Flexbox/Grid.
  • Semantic HTML Trade-offs
    • Overused div before understanding accessibility benefits of section, nav, etc.
  • Debugging Frustrations
    • Spent hours fixing tiny issues (e.g., missing closing tag or display: block vs inline-block).
    • Leveraged DevTools to inspect and troubleshoot efficiently.
